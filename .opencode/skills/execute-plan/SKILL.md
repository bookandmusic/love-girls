# 执行计划

执行已审核通过的任务规划文档，按照规划步骤逐步完成任务。

## 前置条件

- 存在已审核通过的任务规划文档
- 任务规划文档位于 `docs/` 目录下
- 文档格式符合 task-plan 生成的模板

## 执行步骤

### 步骤1：定位任务规划文档

询问用户需要执行的任务规划文档路径：
```
请提供任务规划文档路径（docs/ 目录下的相对路径）：
- 例如：guides/2026-02-10-用户认证功能.md
```

### 步骤2：读取并解析规划文档

读取任务规划文档，解析以下内容：
- 任务概述
- 详细步骤列表
- 资源需求
- 风险提示

### 步骤3：逐步执行任务

按照文档中的步骤顺序执行：
- 每个步骤执行前，显示步骤名称和预期结果
- 记录执行状态（成功/失败）
- 遇到错误时，根据风险提示处理

### 步骤4：更新执行进度

实时向用户汇报进度：
```
当前步骤：<步骤序号>/<总步骤数>
步骤名称：<步骤名称>
执行状态：执行中/已完成/失败
```

### 步骤5：标记任务完成

所有步骤执行完毕后：
- 汇总执行结果
- 提示用户任务已完成
- 可选：删除或归档已完成的规划文档

## 使用示例

```
请执行 execute-plan skill
```

## 错误处理

| 错误情况 | 处理方式 |
|---------|---------|
| 文档不存在 | 提示用户检查路径，重新输入 |
| 步骤执行失败 | 记录错误，根据风险提示处理，或询问用户 |
| 缺少依赖 | 提示用户安装必要依赖后重试 |

## 注意事项

- 必须有用户确认已审核的规划文档才能执行
- 严格按照文档步骤执行，不随意跳过或修改
- 实时向用户汇报进度和状态
- 遇到错误及时处理或询问用户
