# 修复评论删除计数错误 计划

> 本文档记录修复评论删除后"展开更多评论"显示错误的问题。

## 概述

**创建时间**: 2025-03-31
**优先级**: 高
**影响范围**: frontend/client 评论组件, backend 评论删除逻辑

---

## 需求描述

用户报告：在动态页面删除评论后，即使只剩一条评论，页面仍然显示"展开更多评论"，点击后没有效果。

---

## 问题分析

### 后端问题（根本原因）

`CommentRepo.Delete` 方法只删除单条评论，没有级联删除子评论：

```go
func (r *CommentRepo) Delete(ctx context.Context, id uint64) error {
	return r.db.WithContext(ctx).Delete(&model.Comment{}, id).Error
}
```

删除父评论后，子评论变成"孤儿"记录（`parent_id` 指向不存在的评论），但仍被 `total` 计数。

API 返回示例：
- `comments` 只有 1 条
- `total` 却是 2（包含孤儿子评论）

### 前端问题（已修复）

前端 `handleCommentDeleted` 的 `total.value--` 只减 1，但实际可能删除多条评论。

---

## 实现方案

### 步骤一：修复后端删除逻辑（核心修复）

**文件**: `backend/internal/repo/comment.go`

**改动说明**:
- 修改 `Delete` 方法，使用 `path` 字段递归删除评论及其所有子评论
- 删除条件：`id = ? OR path LIKE ?`（匹配自身及所有路径前缀匹配的子评论）

**修复代码**：

```go
func (r *CommentRepo) Delete(ctx context.Context, id uint64) error {
	comment, err := r.FindByID(ctx, id)
	if err != nil {
		return err
	}
	return r.db.WithContext(ctx).
		Where("id = ? OR path LIKE ?", id, comment.Path+"/%").
		Delete(&model.Comment{}).Error
}
```

### 步骤二：修复前端计数逻辑（已完成）

**文件**: `frontend/client/src/components/comment/CommentList.vue`

已在删除评论前递归计算被删评论及其子评论的总数，正确更新 `total`。
