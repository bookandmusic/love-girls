# 页面切换骨架屏优化计划

> 本文档记录客户端页面切换体验优化的实现计划。

## 概述

**创建时间**: 2026-03-31
**优先级**: 高
**影响范围**: frontend/client 路由、页面组件、UI组件

---

## 需求描述

当前客户端存在以下问题：
1. 点击菜单后需要等待异步路由守卫完成才能切换页面
2. 页面切换时使用全局 Loading 遮挡，用户看不到任何内容

期望优化为：
1. 点击菜单后立即切换页面（使用缓存状态）
2. 首次启动时等待状态加载完成（必须阻塞）
3. 每个页面先显示骨架屏/占位动画
4. 数据请求成功后渲染真实数据

---

## 实现方案

### 步骤一：创建全局状态刷新 composable

**文件**: `frontend/client/src/composables/useGlobalRefresh.ts`

**改动说明**:
- 创建全局状态刷新 composable
- 提供 `initGlobalState()` 方法：首次启动时调用，阻塞加载必要状态
- 提供 `startPeriodicRefresh()` 方法：启动定时刷新（仅刷新 systemInfo）
- 提供 `isReady` 状态：标记全局状态是否就绪
- auth 状态只在首次加载验证，不频繁刷新

### 步骤二：在 App.vue 初始化全局状态

**文件**: `frontend/client/src/App.vue`

**改动说明**:
- 应用挂载时调用 `initGlobalState()` 阻塞加载状态
- 状态就绪后启动 `startPeriodicRefresh()` 定时刷新
- 首次加载时显示启动 Loading，就绪后隐藏

### 步骤三：优化路由守卫（完全移除阻塞）

**文件**: `frontend/client/src/router/index.ts`

**改动说明**:
- 路由守卫移除所有异步等待，改为同步检查
- 直接读取 `authStore.isAuthenticated` 判断认证
- 直接读取 `systemStore.initialized` 判断初始化
- 如果状态未就绪（首次启动），由 App.vue 层面阻塞，路由不做额外处理

### 步骤四：创建通用骨架屏组件

**文件**: `frontend/client/src/components/ui/SkeletonLoader.vue`

**改动说明**:
- 创建通用骨架屏组件，使用 CSS 动画实现脉冲效果
- 提供灵活的插槽设计，允许自定义骨架布局
- 基础样式：灰色渐变背景 + 脉冲动画 + 圆角

### 步骤五：创建各页面骨架屏子组件

**文件**: 
- `frontend/client/src/views/components/HomeSkeleton.vue`
- `frontend/client/src/views/components/AlbumsSkeleton.vue`
- `frontend/client/src/views/components/MomentsSkeleton.vue`
- `frontend/client/src/views/components/PlacesSkeleton.vue`
- `frontend/client/src/views/components/AnniversariesSkeleton.vue`

**改动说明**:
- 每个页面创建独立的骨架屏组件，模拟真实内容布局
- 使用 SkeletonLoader 作为基础构建块
- 遵循各页面的实际 UI 结构

### 步骤六：移除全局 Loading

**文件**: `frontend/client/src/layouts/DefaultLayout.vue`

**改动说明**:
- 移除 `<LoadingSpinner v-show="uiStore.loading" />` 全局遮挡
- 保留 LoadingSpinner 组件用于首次启动认证验证场景

### 步骤七：优化页面数据加载策略

**文件**: 
- `frontend/client/src/views/HomeView.vue`
- `frontend/client/src/views/AlbumsView.vue`
- `frontend/client/src/views/MomentsView.vue`
- `frontend/client/src/views/PlacesView.vue`
- `frontend/client/src/views/AnniversariesView.vue`

**改动说明**:
- 移除 `uiStore.setLoading(true)` 全局 loading 调用
- 添加 `isLoading` 本地状态控制骨架屏显示
- 页面模板中添加骨架屏条件渲染：`<XxxSkeleton v-if="isLoading" />`

### 步骤八：优化认证 Store 缓存机制

**文件**: `frontend/client/src/stores/auth.ts`

**改动说明**:
- 添加 `authChecked` 标志记录是否已检查过认证状态
- `checkAuthStatus` 方法在已检查时直接返回缓存结果
